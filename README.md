# 编程题集

## 第1题 计算快递包裹运费

### 题目描述
请编写程序，计算快递包裹的运费。快递公司根据包裹的重量和体积计算运费，具体规则如下：运费 = 基础运费 + 超重费 + 超体积费。
基础运费：5公斤（含）、0.1立方米（含）以内，固定为10元。
超重费：如果包裹重量超过5公斤，超出重量按照2元/公斤计费。
超体积费：如果包裹体积超过0.1立方米，每超0.01立方米加收1元。

### 输入格式
两个数值，分别表示包裹的重量（公斤）和体积（立方米），最多保留小数点后三位。

### 输出格式
运费数值（元），保留小数点后两位。

### 测试用例
[测试用例1]
输入：
4   0.11
输出：
11.00

[测试用例2]
输入：
5.025   0.2
输出：
20.05

## 第2题 判断易混淆数

### 题目描述
给定一个正整数 N（N的位数最高不超过200位），判断它是否是一个“易混淆数”。易混淆数的判定规则为：将该整数整体旋转180°，如果旋转后能得到一个有效的整数，且与原整数不同，则称之为易混淆数。例如：189整体旋转后变成681，即数字0,1,6,8,9旋转180°后分别变为0,1,9,8,6；数字2,3,4,5,7旋转180°后无法得到有效的数字。11616旋转后变成91911；1234旋转后得不到一个有效的整数。

### 输入格式
一个整数N，表示需要判断的数字。

### 输出格式
若N为易混淆数，输出"YES"，否则输出"NO"。

### 测试用例
[测试用例1]
输入：6
输出：YES

[测试用例2]
输入：916
输出：NO

[测试用例3]
输入：8101
输出：YES

[测试用例4]
输入：25
输出：NO

## 第3题 校验公民身份证号码

### 题目描述
公民身份证号码编码规则如下图所示：
image.png
其中，最右边一位校验码采用ISO 7064:1983.MOD 11-2 校验算法对左边17位进行校验。具体校验公式如下：
∑(i=1 to 18) (a_i × W_i) ≡ 1 (mod 11)
其中，i表示身份证号码字符从右至左包括校验码字符在内的位置序号；a_i表示第i位置上的号码字符值，a_1是校验码的值；W_i是第i位置上的加权因子，加权因子的计算公式为：W_i = 2^(i-1) (mod 11)。
根据上述校验算法，校验码的产生规则其实是左边17位字符值与对应的加权因子的乘积的累加和再模11之后得到的值（余数为10的时候用X代替）。请编写程序校验输入的身份证号码是否符合编码规则。

### 输入格式
第一行为测试样例的个数，后面若干行，每一行为一个身份证号码。

### 输出格式
若干行，校验通过，输出True，否则输出False。

### 测试用例
输入：
2
110108201712215919
110108201712215910
输出：
True
False

## 第4题 最大群聊问题

### 题目描述
在一个社交网络中，每个用户都有自己的朋友圈。现在需要设计一个程序，帮助现有的用户建立一个群聊，满足条件：1、群聊中的任意两个成员必须是直接朋友或至少拥有一个共同的朋友；2、群聊的规模尽可能大（即包含尽可能多的用户）。

### 输入格式
第一行是用户数目 N（编号分别为0至N-1）。然后第2至第N+1行表示用户0至N-1的直接朋友列表，每一行以-1表示结束。注意：直接朋友关系是双向的，输入中不会违反此约束。

### 输出格式
第一行输出最大群聊的规模。
第二行输出最大群聊的成员。

### 测试用例
[测试用例1]
输入：
5
1 2 -1

[测试用例2]
输入：
5
1 -1
0 2 3 -1
1 4 -1
1 -1
2 -1
输出：
4
0 1 2 3

## 第5题 最短路径规划

### 题目描述
给你n个顶点，m条无向边，每条边都长度d和花费p，给定一个起点s和终点t，要求输出起点到终点的最短距离及其花费，如果最短距离有多条路线，则输出花费最少的。

### 输入格式
输入n,m，（表示含有n个顶点，顶点的编号为1,2,……,n），然后是m行，每行4个数a,b,d,p，表示a和b之间有一条边，且其长度为d，花费为p。最后一行是两个数s,t;起点s，终点t。（1<n<=1000，0<m<100000，s != t）。

### 输出格式
输出一行，两个数，即从s到t的最短距离及其花费。

### 测试用例
[测试用例1]
输入：
3 2
1 2 5 6
2 3 4 5
1 3
输出：
9 11
